# Портфолио — инструкция по редактированию

Сайт полностью готов к использованию. Все тексты, ссылки и кейсы
хранятся в одном файле — **`js/data.js`**. Именно его нужно
открывать, чтобы что-то изменить. Остальные файлы трогать не нужно.

## Как открыть сайт

Просто откройте файл `index.html` в браузере (двойной клик).
Либо загрузите всю папку на любой хостинг (Tilda, Netlify, Vercel,
GitHub Pages и т.д.) — сайт статический, сервер не нужен.

## Что где менять (всё в файле `js/data.js`)

| Что нужно изменить | Где искать в data.js |
|---|---|
| Имя, роли, описание | `profile` |
| Фотография | `profile.photo` — положите файл в папку `assets` и укажите путь |
| Ссылка на резюме (PDF) | `profile.resumeLink` |
| Telegram / WhatsApp / Email | `contacts` |
| Карточки «Избранные работы» | `featuredWorks` — добавляйте/удаляйте объекты в списке |
| Категории кейсов и работы внутри них | `caseCategories` |
| Текст «Обо мне» | `about.text` |
| Список инструментов | `tools` |

## Как добавить новую работу в категорию кейсов

Найдите нужную категорию в `caseCategories` (например `id: "commercial"`)
и добавьте новый объект в её список `works`:

```js
{
  title: "Название работы",
  description: "Короткое описание",
  type: "PDF", // как подписать тип файла: PDF, DOCX, Figma, Google Docs...
  link: "https://ссылка-на-файл",
}
```

Ничего в HTML или CSS менять не нужно — карточка появится на сайте
автоматически.

## Как добавить новую категорию кейсов

Скопируйте один из существующих блоков внутри `caseCategories`
(фигурные скобки `{ ... }` целиком), вставьте его в конец списка
и измените `id`, `icon`, `title`, `description`, `tags` и `works`.

## Как заменить фотографию

1. Положите свою фотографию в папку `assets` (например `assets/photo.jpg`).
2. В `data.js` найдите строку `photo: "assets/photo-placeholder.svg"`
   и замените путь на `assets/photo.jpg`.

## Как заменить иконки инструментов на настоящие логотипы

По умолчанию инструменты показаны аккуратными плашками с инициалами
(это сделано специально, чтобы не использовать внешние ссылки на
иконки, которые могут перестать работать).

Если хотите использовать свои логотипы:
1. Положите файлы иконок в `assets/icons` (например `assets/icons/figma.svg`).
2. В `data.js`, в списке `tools`, у нужного инструмента укажите
   `image: "assets/icons/figma.svg"` — плашка автоматически покажет
   картинку вместо инициалов.

## Структура файлов проекта

```
index.html        — разметка страницы
css/style.css      — все стили и цвета
js/data.js         — ВЕСЬ контент сайта (редактируется здесь)
js/script.js       — логика отображения и интерактивность (менять не нужно)
assets/            — фото и иконки
```
